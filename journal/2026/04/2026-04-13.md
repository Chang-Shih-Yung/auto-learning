# 2026-04-13 學習日誌

## Claudraband：讓 Claude Code 變成可程式化服務

Show HN 上出現了一個 113 stars 的新工具 [Claudraband](https://github.com/halfwhey/claudraband)（30 commits、6 releases），設計目標是讓 Claude Code session 在關掉終端後繼續活著——可以透過 HTTP daemon 從外部腳本遠端控制 Claude Code、恢復中斷的工作，或讓多個 session 並行處理不同任務，TypeScript 93.6% 撰寫，附三個可執行範例。

架構上，Claudraband 把 Claude Code 的 TUI 包在一個受控 terminal（預設 tmux）裡，對外同時暴露 HTTP daemon 和 ACP server 兩個介面，讓 Claude Code 從「要人在旁邊盯著」的互動工具，變成一個可以在排程任務裡靜默執行的服務。核心思路和 Thin Harness 架構一致：把 Claude Code 當成可程式化的執行引擎，上層的 orchestration 邏輯由外部腳本控制。

以前 Claude Code 工作流是「開了就要一口氣做完」——斷線、換裝置、context 就消失了。這個工具提出的方向是把 session persistence 和 HTTP 控制分離出來，讓 AI coding 工具可以被 CI pipeline 排程驅動，而不只是互動式使用。這個設計模式值得關注，等工具成熟度提升後可以深入評估。

> ⚠️ 目前 113 stars，剛首發的新工具，尚未通過廣泛社群驗證。概念設計值得參考，但建議等 stars 累積到 500+ 後再考慮實際整合。

## UC Berkeley：主流 AI Agent Benchmark 全面淪陷

UC Berkeley RDI 中心發布了一篇讓整個 AI 評估圈震動的研究：他們對 8 個最常被引用的 AI agent benchmark 全部 exploit 成功——[Terminal-Bench](https://rdi.berkeley.edu/blog/trustworthy-benchmarks-cont/)（89 任務）100%、SWE-bench Verified（500 任務）100%、SWE-bench Pro（731 任務）100%、WebArena（812 任務）~100%、GAIA（165 任務）~98%、OSWorld（369 任務）73%。你現在看到的那些排行榜分數，可以被任何人複製出來。

漏洞集中在 7 個設計缺陷：evaluator 和 agent 沒有隔離、答案夾帶在測試檔案裡、不安全的 `eval()` 呼叫、LLM judge 沒有防護、字串 matching 太脆弱。Terminal-Bench 靠 binary wrapper trojan；SWE-bench Verified 靠 pytest hook injection；GAIA 直接讀公開答案檔。這些 exploit 都不需要神奇的技術，只需要讀懂 benchmark 的實作邏輯。

以前你選 AI 工具會看 SWE-bench 排行榜，相信「分高代表能力強」；現在你知道那個分數可以在不具備真實能力的前提下達到 100%。未來評估工具的方法只有一個：自己在真實 task 上測試，或選那些公開「對抗測試方法論」的評估系統。benchmark 排行榜退場，個人驗收標準升格。

## Next.js v16.2.3：CVE-2026-23869 安全 Patch 需主動升級

Next.js 在 4 月 8 日發布了 [v16.2.3](https://github.com/vercel/next.js/releases)，這是一個 backport 安全修補版本，修復了 CVE-2026-23869 安全漏洞，同時解決了 styled-jsx 渲染並發問題（高負載下樣式會靜默丟失）、強化了 ISR revalidation 錯誤回報，並修了 HMR 在 manifest.ts 情況下中斷的問題。這些都不是新 feature，但 CVE 等級的安全問題不能等 Vercel 自動處理。

這個 patch 修的是 styled-jsx 在並發渲染時的 race condition——多個請求同時觸發 SSR 時，某個 request 的樣式可能被另一個請求的上下文覆蓋，導致 HTML 輸出缺少 CSS。v16.2.3 用請求隔離的方式解決了這個問題，同時把 CVE-2026-23869 的安全修補一起 backport 進來（細節未完整披露，但修補已包含）。

以前 minor patch 可以慢慢等 Dependabot 提 PR → CVE 等級的修補需要今天就處理。更重要的是，這提醒你把 Claude_auto 的 Next.js 版本從 `^16.x` 的自動升級改成 `~16.2.x` 的鎖定 patch，搭配 `npm audit` 接進 CI pipeline，安全問題自動通報而不是被動等待。

## cirruslabs/tart：Apple Silicon 上的原生 macOS VM CI

[cirruslabs/tart](https://github.com/cirruslabs/tart) 今天在 GitHub Trending 出現，+124 stars（總計 5,445），Swift 撰寫。它讓你在 M 系列 Mac 上用 Apple 的 Virtualization framework 原生跑 macOS 和 Linux VM——不是 Rosetta 模擬、不是 x86 emulation，是 ARM 原生速度，搭配 `tart run` 一行指令起 VM，做 CI job 隔離測試。

Tart 底層用 Apple Virtualization.framework（macOS 12+ 原生支援），每個 VM 是獨立的磁碟 image，可以 clone、snapshot、push 到 OCI registry。這和 Docker container 的核心差異在：tart VM 可以跑完整的 macOS kernel，代表你可以測試需要 Xcode、Metal API、System Extension 的東西——這些是 container 做不到的。

以前 macOS CI 要靠 GitHub Actions 的 macOS runner（按分鐘計費、常排隊）或租 AWS Mac instances（$1.083/hr）→ 現在你的 MacBook 就是 CI server，VM 跑完刪掉，零成本。對正在學 containers 的 Henry 來說，Tart 是很好的對照組：VM 提供完整 OS 隔離，container 提供輕量 process 隔離，兩者的取捨從這裡開始看得最清楚。

## 今日總結

今天的內容圍繞一個核心洞察：**工具的可信度需要自己驗證，不能外包給排行榜或 patch 週期**。

三個方向：
1. **Claude Code 延伸**：Claudraband 把 Claude Code 從互動工具變成可程式化服務，你的排程任務工作流可以直接受益，值得這週試跑一個 session persistence 的 poc。
2. **AI 評估視角**：Berkeley 的研究改變了「選工具看 benchmark」的邏輯——你在 Nexus 元件庫 AI 生成的驗收標準（pixel perfect、token 對齊、WCAG），才是真正有意義的指標。
3. **維護意識**：Next.js CVE 提醒你 `npm audit` 應該進 CI pipeline，不只靠 Vercel 的自動部署保護生產環境。
