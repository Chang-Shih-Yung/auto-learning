## Lemonade by AMD — 開源本地 LLM 伺服器 v10.0，NPU + GPU 聯動 55 t/s@12W，Linux 正式支援

在搭載 Ryzen AI 300 的 Windows / Linux 機器上，Lemonade 可以用 NPU（神經處理器）跑 Llama 3.2-1B，達到 **55 tokens/秒、耗電僅 12W**；8B 模型在 iGPU 上達到 17–20 t/s；prefill 速度（處理輸入提示詞）達約 **400 t/s**——加上 FastFlowLM 0.9.35，NPU 的 context window 可開到 **256,000 tokens**。整個後端只有 2MB，OpenAI API 相容，把現有 AI 工具的 endpoint base URL 改成 `localhost`，不改任何程式碼就能切換到本地模型。Lemonade 10.0 在 2026 年 3 月 11 日正式加入 Linux NPU 支援。

Lemonade 的核心差異是統一了三種推理後端（llama.cpp、ONNX Runtime、Ryzen AI），根據硬體自動路由到最快的路徑。Ryzen AI 300 的特殊之處在於它有獨立的 NPU 晶片——NPU 負責 prefill（把提示詞計算成 KV cache，速度遠快於 GPU），iGPU 負責 decode（逐 token 生成），兩個晶片並行，效率比單用 GPU 更高、耗電更低。OpenAI API 相容讓 Lemonade 能直接插進任何已有 AI 整合的工具鏈，不需要適配層。

以前 AMD NPU 是「花錢買了但沒軟體用」的硬體——所有本地 LLM 工具都針對 Nvidia GPU 或 Apple Silicon 優化，AMD 用戶只能退回 CPU。現在 AMD 自己下場：Lemonade 官方支援 NPU + iGPU hybrid，Linux 也通了。55 t/s、12W——大約一個小燈泡的耗電——代表 AI agent 可以 24 小時在本地待機，不用擔心電費。對想讓 agent 跑在本地、資料不出機器的場景，AMD 生態現在有了可以認真考慮的選項。

## 2026 年電子郵件防爬蟲最新實測 — 426 個爬蟲驗證，純 JS 字串拼接達 100% 防護

如果你的 Angular 或 Next.js 頁面需要顯示聯絡 email，這個研究直接給你答案：對 **426 個**純文字爬蟲和 **399 個** mailto 鏈結爬蟲實測，**JS 字串拼接**（把 email 在前端拼合）和 **JS 使用者互動觸發**（點擊才顯示）對兩種爬蟲都達到 **100% 防護**，使用者體驗完全不受影響——沒有「把 [at] 換成 @」的提示，顯示的就是正常可點擊的 email。而把 email 直接寫進 HTML 原始碼？保護率 **0%**，爬蟲直接帶走。HTML entities 方案（把 @ 換成 `&#64;`）可達 **95% 防護**，但對能執行 JavaScript 的智慧爬蟲無效。

研究發現 email 爬蟲分兩類：只抓 HTML 原始碼的簡單爬蟲，和會渲染完整頁面再抓文字的智慧爬蟲。HTML 技巧只能攔截前者——智慧爬蟲看到的是渲染後的 DOM，HTML entities 在它眼中就是普通的 @ 符號。JS 方法的防護原理是：爬蟲必須完整執行你的 JS 邏輯才能取得 email，而這個成本讓大多數商業爬蟲直接放棄。

```js
// 實作範例：onMount 拼接，對使用者完全透明
const email = ['contact', '@', 'example', '.com'].join('');
```

以前前端工程師在「email 防爬蟲」和「正常 UX」之間只能二選一：直接顯示被爬走，或者「contact [at] example [dot] com」破壞使用者體驗。這份 2026 年實測確認：**JS 字串拼接打破這個二分法**——100% 防護 + UX 零影響，就是幾行程式碼的事。任何有聯絡資訊的企業網站，今天就能套進 Angular `ngOnInit` 或 React `useEffect` 裡，立刻生效。

## 今日總結

**本地 AI 工具鏈標準化：** Lemonade 的 OpenAI API 相容設計揭示了本地 AI 生態正在走向標準化——不論是 Ollama（Mac）、Lemonade（AMD）還是 LM Studio，都對齊同一個 API 介面。這意味著「雲端 API vs 本地模型」的切換成本越來越低，agent 工具鏈可以不綁定任何單一 provider。

**前端安全盲區：** Email 爬蟲防護是一個常被忽略的前端安全細節——沒有 XSS 那麼顯眼，但後果實際（垃圾郵件、競業爬取聯絡名單）。2026 年的實測確認了最佳實作：JS 拼接，100% 防護，UX 不動。這可以是 Nexus 元件庫的一個微小但有用的貢獻。
