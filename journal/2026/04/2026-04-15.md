## Claude Code v2.1.108 — `/recap` 指令與 Prompt Caching 升級

你現在可以在切換回一個暫停的 Claude Code session 時，直接輸入 `/recap`，讓模型在幾十秒內重新整理這個 session 的核心脈絡——不需要自己翻回去找上次做到哪裡。同時，透過設定 `ENABLE_PROMPT_CACHING_1H` 環境變數，你的 API 呼叫可以把 system prompt 快取延長到 1 小時（原本是 5 分鐘）。對 Henry 這種跑長 session 或 scheduled task 的用法，可以顯著降低 token 費用；同一個 session 的 context 不需要每 5 分鐘重新付費。另外，`/undo` 現在是 `/rewind` 的 alias，語法更直覺。

v2.1.108 最大的架構調整是讓 model 能夠「主動發現並呼叫」內建的 slash commands（`/init`、`/review`、`/security-review`），透過 Skill tool 作為橋接層。這代表未來 Claude 不只是執行你下的指令，而是在理解任務後自己決定要不要呼叫 `/review` 或 `/security-review`——agent loop 從「被動執行」升級成「主動選擇工具」。記憶體方面，語法高亮改成按需載入（on-demand language grammar loading），對開 10+ 檔案的 session 特別有感，記憶體佔用顯著降低。

以前切換 session 的成本很高——你必須自己花幾分鐘讀回上次的脈絡，或是在同一個 session 裡撐很久不敢關。現在 `/recap` 把這個成本歸零：關掉 → 隔天繼續 → 輸入 `/recap` 重建上下文。加上 1 小時快取，長跑的 scheduled agent（就像每天執行這個 learning session）的 API 成本結構會有感改變。這不是一個新功能，這是把 session 的「切換摩擦」系統性消除。

## Boneyard — 從真實 DOM 自動生成 Pixel-Perfect Skeleton Screen（4.8k stars）

你在 Vue 或 React 專案裡跑一次 `npx boneyard-js build`，Boneyard 會快照你的真實 DOM（用 `getBoundingClientRect()`），生成一份 `.bones.json`，之後每個 `<Skeleton>` 元件都能自動從這份 JSON 渲染出與實際 layout 完全對齊的載入骨架——不是手刻的近似值，是 pixel-accurate 的 DOM 複製品，在每個斷點都準確，包括你的 PDA 裝置尺寸。Vite 專案下甚至不需要手動重跑：dev server 啟動時自動重新快照，HMR 更新時跟著更新。

傳統的 skeleton 做法是手動估版面：你猜一個 300px 寬的區塊，實際上可能是 324px，然後骨架和內容之間有明顯的跳動（layout shift，CLS 飆升）。Boneyard 反過來——它先讓瀏覽器渲染真實頁面，用 `getBoundingClientRect()` 抓每個元素的精確座標，把這些數字凍結成靜態 JSON。Runtime 不需要重跑 layout engine，直接讀座標繪製骨架，CLS 降到接近零。`.bones.json` 的格式跨框架統一，React、Vue、Svelte 輸出同一份 JSON，理論上 Angular 也能接入。

以前替 EQS 或 MES 這類複雜表單加 skeleton screen，要花半天手刻每個欄位的估算尺寸，改版後還得重刻——這個維護成本讓骨架屏在企業系統裡常常被跳過。現在一次 `npx boneyard-js build`，`.bones.json` 自動跟著你的 DOM 走，改版後重跑一次就更新。骨架屏的維護成本從「每次改版都要重刻」變成「零額外成本」，企業系統的 loading 體驗沒有理由再繼續省這一步。

## opencode — 100K Stars 的開源 Terminal AI Coding Agent

你可以在 terminal 裡用 opencode 把 Claude Max 訂閱直接路由進去——用你已經付費的 Claude 模型，但透過一個更輕量的 TUI 介面操作，不需要開另一個 Claude Code session。opencode 支援 75+ 模型（Claude、GPT-5、Gemini、本地 Ollama），切換語法一行搞定，特別適合在 Claude Code 遇到 rate limit 時有個立即可用的備援，不中斷工作流。它用 Go 寫成，binary 輕量，沒有 Node.js runtime 依賴。

opencode 核心設計是「model-agnostic router」：它不綁定任何一家 API，而是把你的指令翻譯成各家 LLM 的 API 格式，再把回應統一成同一套 TUI 呈現。100K stars 的爆炸成長（兩週內增加 18,000 stars）來自早期發現者分享「Claude Max 可以路由進去」的技巧——讓原本只能在 Claude.ai 網頁用的 Max 訂閱，延伸到所有支援 opencode 的 terminal 工作流。原生支援 MCP，代表你的 Nexus MCP Server 可以接進去直接使用。

以前你的 AI coding workflow 強綁在特定工具（Claude Code 或 Cursor）上——換工具就要換訂閱計畫、換 API key 設定、換工作習慣。現在 opencode 做成一個通用路由層：訂閱計畫不變，工具層可以按任務切換。跑 scheduled agent 用 Claude Code，做 quick interactive 用 opencode，兩個都吃同一個 Claude Max 訂閱，沒有重複付費。離線場景（rate limit 時或沒網路時）可以自動切到 Ollama 本地模型，打造零中斷的 AI coding pipeline。

## 今日總結

今天的三篇核心洞察收斂在同一個主題：**AI 工具正在系統性消除開發摩擦**，而且消除的方式都不是「加功能」，而是把一個你反覆承受的成本歸零。

- **Claude Code /recap** 歸零的是「切換 session 的認知成本」——從你自己重建脈絡 → 模型幾十秒整理好
- **Boneyard** 歸零的是「骨架屏的維護成本」——從每次改版手刻 → `npx` 一次，DOM 快照自動更新
- **opencode** 歸零的是「AI 工具的鎖定成本」——從訂閱綁定工具 → 訂閱路由到任意 terminal 工作流

**可以應用的行動：**
1. 在 Claude_auto 的 scheduled session 設定 `ENABLE_PROMPT_CACHING_1H`，降低每日 session 的 token 費用
2. 在 Nexus 或 Claude_auto 的 Next.js 前端，試跑 Boneyard 自動生成骨架屏，驗證 `.bones.json` 格式
3. 安裝 opencode 作為 Claude Code 的備援，設定 Claude Max 路由，讓 rate limit 不再是工作中斷點
