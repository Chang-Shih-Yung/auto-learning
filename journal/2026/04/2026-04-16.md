## Claude Code v2.1.110 — TUI 全屏模式與行動推播通知

你現在可以在 Claude Code 裡輸入 `/tui` 或設定 `tui: true`，切換到全螢幕 TUI 模式——渲染方式從串流輸出改為 framebuffer 刷新，完全消除捲動時的閃爍問題。同時 v2.1.110 加入了 Push Notification 工具：當你把 Remote Control 的「Push when Claude decides」打開，Claude 會在判斷某個步驟完成或需要你介入時，主動送推播到你的手機——不需要盯著 terminal 等結果。另外 `/focus` 是獨立出來的新指令，取代原本混在 `Ctrl+O` 裡的邏輯，讓 Focus View 的觸發更直覺；`Ctrl+O` 現在專職切換 verbose transcript。

v2.1.110 的設計哲學是「減少 Claude Code 對你注意力的要求」。/tui 解決的是長時間盯著滾動輸出的視覺疲勞——全螢幕渲染讓輸出變得像文件一樣穩定，而不是一行行閃過。Push notification 更進一步：讓 agent 在做複雜任務（例如跑你的 Claude_auto scheduled session）時，你可以把手機放一邊做別的事，Claude 完成後或卡關時會自己找你。`/focus` 的分離讓 UI 控制更符合認知直覺——一個指令管一件事。

以前跑長任務時你必須選擇：要麼守在螢幕前等，要麼走開但擔心任務卡住沒人知道。v2.1.110 的推播通知把這個二選一變成「走開，讓 Claude 通知你」——Claude 不只是執行者，現在也是主動的溝通方。加上 TUI 全螢幕讓你回來時能快速掃視任務狀態，Claude Code 的「可信度」在長時間任務裡顯著提升。

## Gemini 2.5 Flash — 1M Context 加上可控的思考預算

你可以透過 Google AI Studio 或 Gemini API 呼叫 Gemini 2.5 Flash，傳入一個文字、圖片、音訊、甚至影片的混合 prompt，在最高 1,000,000 tokens 的 context 視窗裡讓它分析——相當於一次丟進去 750 本技術文件。它支援「思考預算」（thinking budget）：你可以設定模型在回答前花多少 token 進行內部推理，從幾百 token 的快速回答，到上萬 token 的深度推理，按任務需要調節，付費也按思考用量計算。

Gemini 2.5 Flash 的核心設計是「混合推理模型」：它在一個模型裡同時包含快速生成路徑和慢速推理路徑，用 thinking budget 參數決定要啟用哪條路徑。這讓它在 MMLU、MATH、視覺推理等 benchmark 上能以低成本逼近 Gemini 2.5 Pro 的表現，特別是在 coding 和 math 任務上。1M context 的技術支撐是 Google 的 RoPE scaling + FlashAttention，讓長文件的 KV cache 不炸記憶體。

以前你要選擇：用快速廉價模型（精度差）還是慢速貴重模型（精度高）。Gemini 2.5 Flash 把這個選擇變成一個 API 參數——同一個模型，用 `thinking_budget` 告訴它要「想多少」，短任務省錢、複雜任務全力以赴。對前端工程師而言，這是第一次能在 API 層精確控制推理深度和費用，而不是靠選不同的模型名稱。

## OpenAI Codex CLI — 官方開源 Terminal Coding Agent

OpenAI 正式發布了 Codex CLI，一個完全在 terminal 裡運行的 AI coding agent，MIT 授權開源，repo 在 `github.com/openai/codex`。你可以用 `npx @openai/codex` 啟動，在任何 Git 倉庫裡執行自然語言指令，Codex 會讀取你的 codebase、計劃修改、寫程式、跑測試，整個流程完全在 terminal 裡不需要開瀏覽器。它支援 Sandbox 模式（執行時網路隔離）和 Full Auto 模式，Full Auto 下不需要每步確認。

Codex CLI 的架構與 Claude Code 平行但有差異：Claude Code 走 Anthropic 的模型鏈，Codex CLI 走 OpenAI 的 codex-mini 模型。兩者都是 terminal-native agent，都支援 MCP，但 Codex CLI 在沙盒隔離上做得更嚴格——預設啟用 `--sandbox exec-network-disabled`，阻止 agent 在執行任意 shell 指令時往外發 HTTP 請求。這對企業合規場景（不想讓 AI 在 CI/CD 管道裡往外打流量）有實際意義。

以前 OpenAI 的 coding agent 只活在 ChatGPT 網頁介面裡——你要打開瀏覽器、貼程式碼、等回應，完全脫離 terminal 工作流。Codex CLI 讓 OpenAI 的模型直接住進 terminal，與你的 Git、CI、測試腳本並排工作。對已經在用 Claude Code 的開發者，這是第一個真正能拿來橫向比較的競品——同樣的任務，跑看看哪個代理的計畫更準確、哪個修改更少引入 regression。

## 今日總結

今天三個工具都在搶同一個位置：**terminal 裡的 AI 主力工具**。Claude Code 的 TUI 模式讓長任務更可信、Gemini 2.5 Flash 帶來可調的推理深度、Codex CLI 讓 OpenAI 正式入場 terminal 戰場。

**核心洞察：** 2026 年的 AI coding 競爭不再是「哪個模型比較聰明」，而是「哪個工具在你的 terminal workflow 裡的摩擦最少」。Claude Code 靠 Remote Control + Push notification 建立「放心走開」的信任感，這是純粹的工具設計優勢，不是模型能力優勢。

**可以應用的行動：**
1. 啟用 Claude Code Remote Control 的 Push notification，讓 Claude_auto session 完成後主動通知你
2. 用 `npx @openai/codex` 在 Nexus 倉庫試跑一個簡單任務，實際感受與 Claude Code 的差異
3. 在需要 long-context 分析（如讀大型 codebase）時，評估 Gemini 2.5 Flash API 作為補充
