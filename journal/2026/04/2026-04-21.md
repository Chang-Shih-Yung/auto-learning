## Claude Code v2.1.116：/resume 快 67%，大 session 不再卡頓

你今天更新 Claude Code，/resume 在 40MB 以上的大型 session 裡快了 67%——以前等 4–5 秒的加載，現在不到 2 秒就能繼續工作。MCP 的啟動也跑更快：改成延遲載入 resource template，第一次連接的等待感幾乎消失。外掛依賴現在自動安裝，不需要手動跑指令；Usage 標籤頁一開就顯示 5 小時和週累計用量，不需要等頁面加載才能看到數字。

背後邏輯是「把非必要的載入推遲到真正需要的時候」。/resume 的優化對應 session 檔案越來越大的現實——你寫得越久，context 越重，以前是工具拖慢你的步調，現在是工具配合你的節奏。MCP deferred loading 則是把「連接時全量讀取」改成「用到哪一個才讀那一個」，對掛載工具越多的 server 效果越明顯。

以前你開一個新的 Claude Code session，第一步是等——等 /resume 重建 context、等 MCP server 準備好、等 plugin 依賴安裝完。現在這三個「等」基本上消失了。你的工作流從「準備 → 等 → 工作」變成「打開 → 繼續工作」。

## ggsql：在 SQL 裡直接寫視覺化，不需要切換到 Python

你已經在學 SQL，現在有一個新的擴充語法讓你在同一個 query 裡完成「取資料 + 畫圖」兩件事。ggsql 讓你寫：

```sql
VISUALIZE bill_len AS x, bill_dep AS y, species AS color FROM penguins
DRAW point
DRAW smooth
```

不需要切換到 R 或 Python，不需要把資料 export 出來再畫圖。一個散點圖加迴歸線，就是這 3 行。由 ggplot2 的作者 Hadley Wickham 與 Thomas Lin Pedersen 主導，2026-04-20 發布 alpha 版。

ggsql 把「Grammar of Graphics」（ggplot2 的設計哲學）移植進 SQL 語法。所有運算都推到資料庫端執行——histogram、boxplot 的統計計算在 server side 完成，你只拿到最終需要畫圖的資料，不是全部 raw data。這避免了「把 100 萬筆資料撈出來再用 Python groupby」的反模式，讓 SQL 成為視覺化的 single source of truth。Rust 實作，支援 Positron、VS Code、Jupyter 環境。

以前你的 SQL 學習路徑是「寫查詢 → export → 開 Jupyter/R → 畫圖」，每次驗證一個假設都要跨兩個環境。ggsql 把這個流程縮成一步，而且留在你熟悉的 SQL 環境裡。目前是 alpha 版，但架構設計已清晰——值得追蹤。

## MCP 最大的缺口：Governance 層的缺失

你在 Nexus 裡建了 MCP Server，讓 Claude 能直接呼叫 Angular 元件生成工具。現在有一篇第一手實作報告告訴你：等你要把 MCP 部署給多個團隊、多個 agent 用的時候，會遇到三個 MCP 本身沒解決的問題——沒有 authentication（誰能呼叫哪個工具？）、沒有輸出限制（50MB+ 的 context 直接回傳怎麼辦？）、沒有 audit log（誰在什麼時間呼叫了哪個 tool？）。作者實測：只要知道 port 號，就能呼叫 server 上的所有工具——沒有任何 access control。

MCP 的設計哲學是「標準化接口」，但沒有標準化治理層，作者的結論是「MCP standardizes the plug, not the power grid」。解法是在 agent 和 MCP server 之間加一層 gateway：allowlist 控制可呼叫的工具、output guardrail 限制回傳大小（超過 5000 字的 README 直接截斷）、audit log 記錄每一次 tool call。更進階的架構是「Virtual MCP Server」——為不同的 agent 預先打包不同的工具子集，不同 agent 看到不同的工具清單，而不是共用同一個 server。

以前你想的是 MCP = 工具整合完成。現在這篇文章讓你看到下一層問題：MCP 解決了「工具怎麼接」，沒有解決「誰能用、用多少、用了什麼」。如果 Nexus 元件庫以後要接多個 agent 或讓多個開發者共用，governance layer 是你提前需要設計的部分，不是出問題了才加的補丁。

## 今日總結

今天四篇文章有一條隱藏的主線：**把複雜度推到正確的層，讓上游的使用者感受不到它**。

Claude Code v2.1.116 把「等待」這件事推遲到真正需要的時候——deferred loading 是這個哲學的工程實作。ggsql 把統計計算推到資料庫端，讓 SQL 使用者不需要知道 Python 的存在。MCP governance gap 則提醒你：你現在感受不到問題，是因為你的 Nexus MCP server 目前只有你一個人用——當第二個使用者出現的時候，governance layer 的缺失才會變得可見。

這三件事的設計洞察可以整合成一條原則：**好的系統設計不是加更多功能，是把摩擦放到最少人感受到的地方**——工具的等待歸零（CC 116）、環境切換歸零（ggsql）、access control 在 gateway 統一處理（MCP governance）。

Qwen3.6-Max 的免費釋出則是一個市場訊號：AI 能力的商品化速度比預期快。頂級推理能力的 API 成本正在走向 $0。你的 Claude_auto 工作流值得預先設計多模型路由，而不是把 Claude 當作唯一選項。
