## Claude Opus 4.7 系統提示詞解密：工具優先，不再反問

Opus 4.7（2026-04-16 發布）比 Opus 4.6（2026-02-05）多了三個行為層面的直接改變：一、當工具可用時，Claude 現在優先呼叫工具（搜尋、查位置、查日曆）來解決模糊性，而不是先問你「你在哪裡？」；二、你說「好了，謝謝」，Claude 不會再試圖延伸對話；三、4.6 系統提示裡明文寫著「Donald Trump 是現任總統，2025-01-20 就職」的行，在 4.7 裡消失了，代表模型的世界知識已可靠延伸到 2026 年初，不再需要外掛補丁。如果你在 Claude Code 裡接上 MCP Server 工具，這個「工具優先」的新行為會讓你的工具被呼叫的頻率顯著上升。

Simon Willison 長期追蹤 Anthropic 公開的系統提示詞 diff，這篇分析記錄了 4.6 → 4.7 的兩個核心調整：Anthropic 把「有工具可用 → 先行動 → 再報告」的觸發順序明確寫進 system prompt，讓 Claude 的決策樹變成「先用工具解決，而不是先問用戶補充資訊」。同時刪除過時的時效性硬編碼資訊，意味著模型本體知識庫的更新節奏已跟上了系統提示的需求。

以前：工具是 Claude 的「被動選項」，你的 MCP Server 只有在 Claude 確定需要時才被呼叫。現在：工具成了 Claude 的「第一反應」，有工具就試，不問用戶先動手。這對設計 MCP Server 的影響是：你的工具設計策略從「提供選項給用戶確認」，轉向「直接執行並回報結果」——工具應該是自給自足的，不要假設 Claude 會事先詢問缺少的參數。

## Show HN: Prompt-to-Excalidraw — Gemma 4 E2B 在瀏覽器裡畫架構圖

打開桌面版 Chrome 134+，用一句話描述你要的圖形（「user → API → database 的 sequence diagram」），Gemma 4 E2B 在 30+ tok/s 的速度下生成 Excalidraw 圖形——完全跑在你的瀏覽器裡，M1 Mac、3.1GB 一次下載，之後零 API 費、零帳號、離線可用。這個 demo 的最大突破是：LLM 輸出的是 ~50 tokens 的 compact code，而不是 raw Excalidraw JSON（~5,000 tokens），100× 的輸出壓縮讓推理速度足以即時交互。產出的圖形可以直接在 Excalidraw 裡編輯，不是截圖、不是 SVG 光柵，是完整的可編輯矢量物件。

這個 demo 的推理引擎跑在 WebGPU（WGSL compute shaders）。TurboQuant 演算法把 KV cache 壓縮 2.4×，讓更長的對話能塞進 GPU 記憶體。Gemma 4 E2B 是 Google 官方開源模型，設計目標是本地邊緣端推理，所以有 4-bit 量化版本在瀏覽器裡跑得動。這個「緊湊 code → JSON 展開」的架構，讓 LLM 的輸出格式從「人類可讀文字」解耦出來，改成「機器可執行的微型 DSL」。

以前：架構圖要開 FigJam 或 draw.io，AI 生成的最多是截圖（不可編輯）。現在：一句話、瀏覽器本地、輸出可編輯矢量圖形，完全離線，零成本。這條技術路線同時說明了 AI 輔助設計工具的下一個方向：不是讓 AI 生成圖片，而是讓 AI 生成可編輯的結構化資料格式（JSON / SVG）。這個「結構化輸出」的設計哲學，和 Figma → JSON → Token 的工作流是同一個思路——讓每一層輸出都是可程式化的，而不是終態的圖片。

## Claudoscope — Claude Code 的本地成本儀表板

Claudoscope 是一個 macOS menu bar app，讓你瀏覽完整 Claude Code session 歷史、跨 session 全文搜尋、按專案追蹤 token 消耗與實際成本（支援 Anthropic 和 Vertex AI 定價）。附帶兩個安全功能：entropy-based credential scanner（自動找 API key、token 洩漏到 session log 的情況）、19 條規則的 config health linter（檢查 CLAUDE.md、skills、hooks 的設定問題）。完全本地運算，MIT 授權，零 telemetry，不上傳任何資料。

App 直接讀取 Claude Code 在本地生成的 session log 檔案（`~/.claude/` 目錄），彙整 input/output/cache token 數，對應 Anthropic 官方定價計算每次 session 的實際成本。Credential 掃描使用 entropy 分析（高隨機性字串特徵 = 可能是 secret key）搭配 regex pattern match，在本地完成，不需要網路。19 條 linter 規則涵蓋 CLAUDE.md 格式、hook 設定衝突、skill 版本鎖定等常見問題。

以前：你只能看 Anthropic 的月帳單彙整，不知道哪個 session 燒了最多 token，也不知道哪個 CLAUDE.md 設定是有問題的。現在：按專案、按 session 拆解成本，知道「今天 daily digest 花了多少 token、Nexus MCP Server 對話每次成本多少」，並且有自動化工具掃描你的設定和 log 裡的潛在資安問題。

## GitHub CLI `gh skill` — AI Agent 技能的套件管理器

GitHub CLI v2.90.0（2026-04-16）加入 `gh skill` 指令，讓你從命令列探索、安裝、更新、發布 agent skills。裝一個 skill，跨工具共用：同一份 skill 在 Claude Code、GitHub Copilot、Cursor、Codex、Gemini CLI 上都能載入。社群 skill 庫（VoltAgent/awesome-agent-skills）目前已超過 1,000 個 skill，覆蓋 TypeScript 開發規範、Next.js 最佳實踐、資安稽核、API 測試等主題。安裝一條指令：`gh skill install owner/repo`，preview 先看內容不裝：`gh skill preview owner/repo`。

Agent skill 的格式是標準化的 markdown + script 組合，遵循開放的 Agent Skills 規範。版本鎖定（`--pin`）透過 git tree SHA 追蹤，比較的是實際內容變化，不是版號。`gh skill update` 只對有真實內容差異的 skill 提示更新，pinned skill 會被跳過。GitHub 官方警告：skills 沒有平台驗證，安裝前務必 `gh skill preview` 看內容，防範 prompt injection 風險。

以前：每個 AI coding tool 有自己的 custom instruction 格式，Claude Code 的 CLAUDE.md 設定沒辦法搬到 Cursor。現在：一次定義 skill，到處共用，並且有版本管理。你在 Nexus CLI 開發的元件使用規範，可以打包成標準化 gh skill 發布，讓任何使用 Claude Code 或 Copilot 的前端工程師一行指令安裝——把 Nexus 元件庫的 AI 使用說明從私有 CLAUDE.md 升級成公開可複用的知識單元。

## Vercel April 2026 資安事件：你的環境變數安全嗎？

Vercel 在 2026-04-19 確認了一起內部系統遭未授權存取的資安事件，目前最高討論度的 HN 文章（509 points）。根據 Vercel 官方公告，潛在洩露的資料包含：未標記為 sensitive 的 environment variables、API keys、NPM tokens、GitHub tokens，以及 Linear 內部工單資料。關鍵行動：進入 Vercel dashboard，把所有含 secret 的 env var（如 `ANTHROPIC_API_KEY`、`GITHUB_TOKEN`、資料庫連接字串）確認已勾選「Sensitive」，並執行一次輪換。

事件根源是一個小型第三方 AI 工具的 Google Workspace OAuth app 遭到更大範圍的入侵，進而波及使用該工具的組織。威脅者聲稱是 ShinyHunters，在駭客論壇上掛出 Vercel 資料並要求 200 萬美元贖金（Vercel 目前沒有透露談判細節）。Vercel 表示：標記為「sensitive」的 env vars 以不可讀取的方式儲存，目前沒有這部分被存取的證據。第三方 OAuth app 的供應鏈攻擊是這次事件的核心機制——即使你的程式碼本身是安全的，平台層的第三方整合仍然是風險面。

以前：「env var 放在 Vercel 應該安全，Vercel 是大平台」。現在：大平台的第三方整合是供應鏈攻擊的入口，sensitive 標記是實際防線，不是可選的設定。未標記為 sensitive 的 env var，在平台遭入侵時等同明文。對部署在 Vercel 的個人專案（Claude_auto、任何帶有 Anthropic API key 的部署），現在是養成 env var 安全衛生習慣的最好時機：標記 sensitive、定期輪換。

## 今日總結

今天的五篇文章圍繞著同一個核心主題：**AI 工具的可控性**。

Claude 4.7 的系統提示詞變化，把「工具先行動」的決策邏輯顯性化——這對 MCP Server 設計者意味著工具需要設計成自給自足的，而不是依賴用戶補充參數。`gh skill` 建立了 agent skill 的跨工具標準，讓 AI 工具的知識變成可版本管理的模組，而不是一次性的 prompt。Claudoscope 把 Claude Code 的成本從「月帳單黑盒」變成「每個 session 的可量化數字」。

Prompt-to-Excalidraw 展示了「AI 輸出可編輯結構化格式」這條設計路線的可行性——100× 的輸出壓縮讓瀏覽器本地推理變成現實，這個思路直接呼應了 Figma → JSON 的設計系統工作流。

Vercel 的資安事件是這週最需要立刻行動的：進 dashboard 確認所有含 secret 的 env var 已標記為 sensitive，今天就做。

可觀察趨勢：AI agent 的「基礎設施層」正在快速標準化——skills、tools、cost tracking、supply chain security——這些基礎設施一旦成熟，agent 的設計重心就會從「怎麼讓 Claude 跑起來」轉向「怎麼設計最好的工具邊界」。
