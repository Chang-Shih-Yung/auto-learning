# 2026-04-05 學習日誌

## 1. Caveman：讓 Claude Code 少說 75% 的廢話，速度快 3 倍

你可以用一個 Claude Code skill（caveman）讓 Claude 的回應從 1180 個 token 壓縮到 159 個——以解釋 React re-render bug 為例，節省了 87%。平均壓縮比例是 75%，視任務複雜度在 22% 到 87% 之間浮動。少了 token，生成速度快了約 3 倍。一份 2026 年 3 月的研究發現，強制模型用精簡語言回應反而讓特定 benchmark 上的準確率提升了 26 個百分點——更少的廢話 = 更準確的答案。

Caveman 的運作方式是在 Claude Code 的輸出層套上語言約束：移除填充詞、客套語、過度解釋，但完整保留程式碼區塊、技術術語、錯誤訊息和 URL。它還有一個 "Caveman Compress" 功能，把 memory 檔（CLAUDE.md、session log）重新壓縮，input token 平均可以削減 45%。整體效果是雙向的：Claude 說的少，你給的也少，整個 session 的 token 消耗大幅下降。

以前你在等 Claude Code 輸出長篇解釋——3 段描述 + 1 段 before/after + 1 段最佳實踐。你需要的只是「這裡改成這樣，因為 X」。Caveman 讓 Claude 說出你需要的那一句，而不是包裝成教學文章。在你的排程任務場景，每天跑的 session 加起來可能省下數千個 token，直接反映在 API 成本上。

## 2. MacBook M4 Pro 跑 Gemma 4：51 tokens/秒，只用 18GB

你可以在一台 MacBook Pro M4 Pro（48GB）上用 LM Studio 本地跑 Gemma 4 26B，速度是 51 tokens/秒，time to first token 約 1.5 秒——夠快到「可以對話」。模型的量化版本（Q4_K_M）只佔 17.99GB RAM，加上預設 48K context 的 KV cache 約 21GB 總用量，在 48GB 機器上有足夠的空餘。Gemma 4 的 MMLU Pro 分數是 82.6%，AIME 2026 是 88.3%，在同等模型規模（26B）裡屬於頂端。

Gemma 4 26B 用的是 Mixture of Experts（MoE）架構：128 個 expert 加 1 個 shared expert，但每個 token 只激活 8 個 expert（相當於 3.8B 的有效參數）。這就是為什麼 26B 的「名義大小」可以跑在 18GB 記憶體裡——你付的記憶體成本對應的是 4B active，不是 26B 全部。實測 GPU 利用率 90%，功耗 23.56W，溫度約 91°C，是 Apple Silicon 在高負載下的正常範圍。

以前在本地跑 26B 等級的模型，你需要大顯卡（32GB VRAM 的 RTX 4090，要價 $1600+）或租 GPU 雲端。現在一台 MacBook M4 Pro 加 LM Studio CLI（`curl -fsSL https://lmstudio.ai/install.sh | bash`，一條命令安裝）就能跑，51 t/s 足夠日常對話和程式碼輔助。而且所有運算在本機，沒有 API 費用、沒有資料外洩疑慮。

## 3. M3 Pro 跑即時語音 + 視覺 AI：2.5 秒延遲，6 個月前需要 RTX 5090

Parlor 是一個開源專案，讓你在 Apple M3 Pro 上做到即時的多模態 AI 對話——你說話，它看到你的螢幕，3 秒內給出聲音回應。實測：語音理解 + 視覺分析 1.8-2.2 秒，回應生成（25 tokens）0.3 秒，語音合成 0.3-0.7 秒，端到端延遲 2.5-3.0 秒。GPU 解碼速度 83 tokens/秒。作者特別指出：6 個月前，這需要一張 RTX 5090 才跑得動。現在 M3 Pro 就夠了。

技術棧是三個本地模型的組合：Gemma 4 E2B 負責理解語音和視覺輸入（multimodal）、Kokoro 負責文字轉語音。全部跑在 LLM.swift，利用 Apple Silicon 的 Metal GPU 加速，不需要 Python 或 Docker 環境。「即時」的關鍵在於流式輸出（streaming）——不是等整段回應生成完再播放，而是一邊生成一邊說，讓感知延遲從 3 秒降到「說完第一句話只需要等 0.3 秒」。

6 個月的軍備競賽壓縮了什麼：以前 RTX 5090（約 $2000 美元）做到 80 t/s 的解碼速度，現在 MacBook M3 Pro（Apple Silicon Neural Engine + Metal GPU 協作）達到 83 t/s。加上 MoE 架構壓低記憶體門檻，原本需要工作站的任務進入了筆電的射程。對 Henry 的 Nexus 元件庫，這意味著「語音驅動的 design token 調整」不再是科幻——你可以說「把主色調調成溫暖橙色」，AI 即時分析你的螢幕並輸出對應的 OKLCH 值。

## 今日總結

今天三篇的共同主題是：**讓 AI 工具更快、更省、更在地**。

- **Caveman** 是「減法哲學」的最佳示範——去掉 75% 的廢話，速度快 3 倍，準確度反而提升 26%。這挑戰了「越詳細越好」的 prompt 迷信。
- **Gemma 4 本地推理**給了你一個具體的 baseline：51 t/s 在 MacBook M4 Pro 上，18GB，零 API 費用。這個組合現在就可以用，不需要等更好的硬體。
- **Parlor** 是三個本地模型拼接成的即時多模態系統，展示了「組合開源工具」的威力——沒有一個組件是新的，但組合起來的使用者體驗是新的。

三篇的底層邏輯：你不需要等 AI 變更強，你需要的是把現有的 AI 用得更有效率。
