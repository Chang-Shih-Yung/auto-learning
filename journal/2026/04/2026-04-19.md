## Claude Code v2.1.114 — Agent Team 多成員並行穩定性修復

這個版本修復了一個影響多代理協作場景的崩潰問題：當 agent team 的某個成員（subordinate agent）請求工具存取權限時，permission dialog 觸發了一個 crash——導致整個 team 的執行中斷。在 v2.1.114 修復之前，你如果用 Claude Code 的 multi-agent 功能（讓多個 Claude instance 協作同一個任務），一旦有 sub-agent 需要跨越權限邊界，整個 session 就會崩潰，無法繼續。v2.1.114 修復後，permission dialog 能正確處理來自 team member 的請求，agent 協作不再因為一個成員的權限請求而全體崩潰。

這個 crash 揭示了 Claude Code 的 multi-agent 架構仍在快速迭代的早期：permission 系統原本是為單個 agent 設計的，當多個 agent 並行運行並可能同時觸發 permission dialog 時，原始的對話框實作沒有正確處理多來源的請求隊列。v2.1.114 的修復讓 agent team 的 permission model 更接近「每個成員的工具請求都可以被獨立審核或自動批准」的正確設計——這是讓 multi-agent workflow 能夠可靠用於生產的前提。一個 agent 等待 permission 確認，不應該讓整個 team 卡住。

這個修復在這個時間點出現有特別的意義：就在 Anthropic 發布 agentic coding 最佳實踐、OpenAI 推出 Codex 大升級的同一週，Claude Code 悄悄補上了讓這些「宏大願景」能實際運作的基礎穩定性。Multi-agent workflow 的潛力——讓多個 Claude 並行分工、各自負責 codebase 的不同部分——在這個 crash 存在的時候是理論上的，修復後才是你可以實際依賴的生產工具。一個穩固的基礎，比十個新功能更有價值。

## CSS Studio — 視覺設計即程式碼的 AI 介面

CSS Studio（cssstudio.ai）是一個 AI-first 的視覺設計工具：你在畫布上拖拉排版和視覺效果，右側即時看到對應的 CSS 程式碼，並可以直接接 AI agent 補完、優化或解釋每一段樣式。它的核心差異點在於「Design by Hand, Code by Agent」——設計的形狀和排版由你決定，CSS 的實作細節（包括 `transform`、`clip-path`、`animation timing function` 等複雜語法）由 AI 自動生成，你不需要背 CSS 屬性名稱或手寫 `cubic-bezier` 的數字。登上 Hacker News 頭版（175 pts），在設計工程社群引發討論。

CSS Studio 的設計哲學是打破「設計工具 vs 程式碼工具」的二元分割：Figma 讓你設計，VS Code 讓你寫程式，兩者之間的 gap 每次都要靠你手動翻譯。CSS Studio 想把這兩個步驟合併成同一個介面——你在設計的同時，程式碼就在旁邊跟著更新，AI 負責確保程式碼在各個瀏覽器裡渲染正確。它不是 Figma 的替代品，而是針對「一次性視覺效果」的專用工具：不需要建立完整的設計系統，只需要一個能快速視覺化並直接輸出可用 CSS 的環境。

> ⚠️ CSS Studio 是商業 Web 工具（cssstudio.ai），無 GitHub 公開 repo，尚無法驗證開源生態數據，使用前請自行確認定價與資料政策。

以前你從 Figma 到程式碼要走一整條鏈：Figma Export → Design Token 格式轉換 → CSS 變數 → 元件套用——即使你在 Nexus 已經把這條鏈自動化到相當程度，手動設計的 one-off 樣式（例如登入頁的特殊視覺效果、互動動畫）還是要自己寫 CSS。CSS Studio 的概念是把這類「一次性複雜樣式」的手寫成本歸零：設計師和工程師在同一個工具裡完成，不需要在兩個軟體之間切換和翻譯。

## 今日總結

今天的主題是 **穩定性為創新護航**：v2.1.114 的 agent team crash 修復看起來只是小更新，但它讓 multi-agent workflow 從「實驗性功能」升級到「可信賴的生產工具」。這種基礎穩定性的補強，才是讓這週 agentic coding 的大願景真正落地的前提。CSS Studio 則代表了 design-to-code 橋接工具的新思路——不是把設計結果匯出，而是讓設計行為本身就是程式碼輸入。

**核心洞察：** 這整週（4/16-4/19）回顧起來，Claude Code 的進化路徑很清晰：從讓 agent 更聰明（Opus 4.7 xhigh、Auto mode），到讓 agent 更可靠（v2.1.114 team crash 修復），到讓 agent 更好用（/tui、push notification、/focus）。「更可靠」這個維度往往比「更聰明」更難做，但對生產環境的重要性更高。

**可以應用的行動：**
1. 在 Nexus 試跑一個 multi-agent team 場景（lead + 2 sub-agents 按 package 分工），確認 v2.1.114 後穩定
2. 用 CSS Studio 試做一個複雜動畫效果，評估生成 CSS 的品質和跨瀏覽器相容性
3. 整理本週（4/16-4/19）的工具清單，更新你的 design-to-code 工具矩陣
