## Claude Code v2.1.111–v2.1.113 — Opus 4.7 xhigh + Auto Mode + 原生執行檔

v2.1.111 帶來了這個版本週期最大的模型控制升級：**xhigh effort level**，坐落在 high 和 max 之間，讓你在需要深度推理但不想付 max 級費用時有個中間檔位。你可以用 `/effort` 指令直接開啟互動式 slider（方向鍵選擇，Enter 確認），或在 `.claude/settings.json` 寫 `"effort": "xhigh"` 持久化設定。同時，**Auto mode 正式向 Max 訂閱者開放**：在 Auto 模式下，Claude Code 自動根據任務複雜度切換模型——簡單問題用 Sonnet 省費用，複雜任務自動升到 Opus 4.7。另外 `/ultrareview` 讓 code review 深度再往上拉一個層級，以往 high effort 會漏掉的邊緣案例，ultrareview 會抓到。同週的 v2.1.113 帶來更安靜但更深層的架構升級：**CLI 改為 spawn 原生 binary**（per-platform 可選依賴），取代原本打包的 JavaScript bundle，冷啟動時間顯著縮短；並開放了 `sandbox.network.deniedDomains` 設定——即使 `allowedDomains` 裡有 wildcard，你仍可精確封鎖特定域名；`/loop` 按 Esc 現在能取消掛起的 wakeup，wakeup 事件在 UI 上清楚顯示為「Claude resuming /loop wakeup」。

xhigh 填補了 high 和 max 之間的明顯性能跳躍缺口：在需要抓 edge case 但不需要拉滿算力的任務（例如 code review、設計系統一致性檢查）上，xhigh 的 cost/performance 比值應該是最優的。Auto mode 讓這個選擇自動化——你不需要每次任務開始前手動調，Claude 自己判斷複雜度。原生 binary 的架構升級讓 Claude Code 在長期維護性上更接近真正的原生應用；`sandbox.network.deniedDomains` 則讓企業用戶可以做精細的網路隔離，而不是只能「全開」或「全關」。

以前 effort 設定只有 low / medium / high / max 四個檔位，medium 到 high 之間有個跳躍，但費用也跟著跳。xhigh 把這個空缺填補了。更重要的是 Auto mode 把「選模型」這件事從手動決策變成系統自動優化——對你的 Claude_auto scheduled session，這意味著抓新聞這類簡單任務自動用 Sonnet、寫 journal 這類創意任務自動升 Opus，整體費用在不降品質的前提下自然優化。

## PocketFlow Tutorial-Codebase-Knowledge — AI 自動把 GitHub Repo 轉成教學文件（12k stars）

你在 terminal 裡執行 `python main.py --repo https://github.com/任何公開倉庫 --language zh`，它會爬取 repo、分析程式碼結構，自動生成一份有章節順序的 beginner-friendly 教學文件，解釋核心概念、資料流、關鍵函式的作用，輸出成 Markdown。整個流程大約幾分鐘，不需要手動閱讀程式碼，特別適合要快速上手一個陌生開源專案，或為自己的 library 自動生成使用者文件。目前 12,217 stars，MIT 授權，Python 實作。

PocketFlow Tutorial-Codebase-Knowledge 是 PocketFlow 框架的應用範例——PocketFlow 本身是一個「100 行 LLM framework」，核心抽象是 Node（步驟）+ Edge（條件跳轉）+ Flow（流程組合）。Tutorial-Codebase-Knowledge 用這三個抽象實作了「分析 → 找關鍵文件 → 理解依賴 → 生成章節順序 → 寫教學」的 multi-step agent pipeline。它先用 LLM 找出 repo 的核心模組（而不是讀全部檔案），再按「從入口到細節」的順序組織教學，讓新手不會一開始就被細節淹沒。這個設計模式——先找架構入口，再往細節延伸——本身就是學習陌生 codebase 最有效的路徑。

以前要理解一個新的開源 library，你得花幾個小時讀 README + 讀原始碼 + 跑範例，然後還是可能漏掉核心架構決策。Tutorial-Codebase-Knowledge 把這個過程壓縮到幾分鐘：你不是讓 AI 替你讀，而是讓 AI 把「閱讀路徑」規劃好，再用你指定的語言（中文）寫成教學。對你在 Nexus 元件庫的開發，它也是自動生成 onboarding 文件的工具——新成員加入時不需要你手動寫教學，跑一次就有。

## 今日總結

今天的核心主題是 **AI coding 工具的智慧分層**：Claude Code 不再是「一刀切」地用最貴的模型跑全部任務，而是開始根據複雜度自動分層——這是 AI coding 工具走向生產成熟的標誌。PocketFlow 則展示了另一個方向：AI 不只是幫你寫程式，而是幫你「理解程式」——把代碼庫轉換成可導航的知識，讓人的學習速度追上 AI 生成程式碼的速度。

**核心洞察：** 12k stars 的 Tutorial-Codebase-Knowledge 爆紅，背後的需求是真實的：AI 生成程式碼的速度遠超人類理解程式碼的速度，這個 gap 正在變成新的技術債。能夠「快速理解陌生 codebase」的工具，和能夠「快速生成程式碼」的工具，在 2026 年同樣重要。

**可以應用的行動：**
1. 在 Claude Code 啟用 Auto mode，觀察一週的模型選擇日誌，找出費用優化空間
2. 對 Nexus 倉庫跑一次 Tutorial-Codebase-Knowledge，評估輸出品質是否能替代手寫 onboarding 文件
3. 注意 `/ultrareview` 在 code review 任務的表現，判斷是否值得列為 Nexus PR 流程的標準步驟
