## Claude Managed Agents：把 agent infra 外包給 Anthropic

Anthropic 正式推出 Claude Managed Agents 公開 beta，你現在可以在 Claude 平台上部署持續執行幾小時的 agent，不需要自己搭 orchestration 架構。內部測試顯示，Managed Agents 的任務成功率比自己寫的 prompting loop 高出最多 10 個百分點，越困難的任務提升越明顯。Multi-agent coordination（agents 把子任務 delegate 給其他 agents）目前進入 research preview。

這背後是 Anthropic 把 agent harness 和 production infra 封裝成平台——context 管理、工具執行 sandboxing、錯誤回復、session tracing 都由平台處理。開發者只需要定義 task、tools 和 guardrails，其餘交給系統。Claude Console 內建 tool call 追蹤和失敗分析，可以直接在瀏覽器裡 inspect 每一步執行。

以前：你要自己處理 session 管理、context 溢位偵測、工具執行錯誤的 retry 邏輯，一個 production-grade agent 要花幾個月。現在：公開 beta 開放，官方說法是「幾天就能 ship」。Coding agents 讀 codebase 並自動開 PR、文件處理 agents 跑金融和法律流程——這些都已有團隊在用了。

## Claude Code v2.1.97：Focus View + MCP 記憶體洩漏修復

更新到 v2.1.97 後，在 NO_FLICKER 模式下按 Ctrl+O 可以切換 Focus View——只顯示 prompt、工具摘要（含編輯行數統計）和回應，去掉所有視覺雜訊。更重要的是，MCP 連線的記憶體洩漏被修掉了：以前每小時累積約 50MB 的 connection buffer，長時間 session 後記憶體會持續增長，現在乾淨了。

Focus View 是針對 long-running agent session 設計的，讓你在工具大量執行時能快速定位結果，不被中間輸出淹沒。MCP buffer 洩漏的根因是 connection buffer 沒有正確釋放——在你跑幾小時的 scheduled task 時，這個問題會默默累積，這次徹底修掉。此外新增 `refreshInterval` 設定，讓 status line 可以定期重新執行指令（例如每 30 秒刷新 git worktree 狀態），不再只靠事件觸發更新。

以前：開著 MCP session 跑 scheduled task，幾小時後記憶體異常，session 可能不穩定。現在：穩定了，加上 Focus View 讓長時間工作流更容易觀察。這次更新還改善了 CJK 語言（包含中文）在 slash commands 和 mentions 的支援——你在 Claude_auto 用的中文指令現在更準確。

## Meta Muse Spark：多 agent 並行推理，用 10 倍少的算力達到頂尖水準

Meta 推出 Muse Spark，一個原生多模態推理模型，搭載稱為「Contemplating mode」的功能：多個 agent 並行推理，互相競爭得出最佳答案。在 Humanity's Last Exam 這個極困難基準上達到 58% 準確率（同等 Gemini Deep Think 和 GPT Pro 的極限推理模式），而整體算力消耗比 Meta 上一代 Llama 4 Maverick 少超過一個數量級（10 倍以上）。

Contemplating mode 的技術核心是讓多個 agent 針對同一個問題各自獨立推理，再由主模型合成結論——這是一種 ensemble + debate 混合架構。效率提升的關鍵在於模型本身的精簡（相同能力用更少參數達成），而不只是算力硬堆。模型支援 tool-use、visual chain of thought，以及跨 agent 的任務分派。目前 private API preview 開放申請。

以前：要達到 HLE 58% 這個等級需要 Gemini 或 OpenAI 的頂尖版本，推理成本極高。現在：Meta 用一個數量級更少的算力達到同等水準，意味著這個等級的推理能力門檻在降低。等 API 正式開放，多 agent 並行推理可能成為 agent 工作流的標準配件，而不是頂尖服務的特例。

## 今日總結

今天的核心主題是 **agent infrastructure 的平台化**。

三件事放在一起看很有意思：

1. **Claude Managed Agents** 把 agent orchestration 從「你自己寫 loop」變成「你定義規則，平台跑 loop」。這是 infra 層的抽象化。

2. **Claude Code v2.1.97** 修掉了 MCP buffer 洩漏——說明目前 agent 工具鏈仍在成熟過程中，但修補速度快（距離上次版本不到一週）。

3. **Meta Muse Spark** 的 Contemplating mode 展示了 multi-agent 並行推理的具體形式，而且用更少算力達到頂尖水準。

**應用點**：Claude Managed Agents 的 public beta 值得直接去試——特別是「coding agent 讀 codebase 並開 PR」這個 use case，和 Henry 在 Nexus 的工作流非常接近。短期可以把 Claude_auto 的 session 邏輯遷移到 Managed Agents 上測試，看看 persistent state 是否能解掉目前 context 管理的手動補丁問題。
