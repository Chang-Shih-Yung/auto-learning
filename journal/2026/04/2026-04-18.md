## Anthropic Engineering Blog：Claude Code Agentic Coding 最佳實踐

Anthropic 工程師在官方部落格發布了〈Claude Code: Best practices for agentic coding〉（614 pts on HN），系統整理了讓 Claude Code 在真實專案跑得好的核心實踐。最關鍵的建議包括：**任務切分**（把大任務分成有明確驗收標準的小步驟，而不是一次丟大指令）、**Context 管理**（把重要背景寫進 CLAUDE.md，而不是每次 session 重新解釋）、**工具使用順序**（Read 優先、確認再 Edit、Write 最後）、**Verification Steps**（讓 Claude 自己跑測試或檢查輸出，而不是相信「我完成了」）。文章特別強調：「Agentic loop 最容易失敗的地方不是模型智商，而是任務邊界不清楚」——Claude 傾向「完成任務」而不是「確認任務目標是否正確」。

這篇文章揭示了 agentic coding 的核心失敗模式：你要求 Claude 做 X，Claude 做了一個「看起來像 X 完成」的狀態，但這個狀態不符合你實際的需求。Anthropic 的建議是在任務設計層就預防這個問題：CLAUDE.md 的正確用法是存放「決策框架」（告訴 Claude 如何做決定），例如「prefer editing existing files over creating new ones」這類的原則性語句，而不是知識敘述。Verification Steps 的設計邏輯是：在 Claude 「認為完成」和你「確認完成」之間加一道強制關卡，讓 agent 自己執行客觀的驗收標準（測試通過、lint 無誤、diff 符合預期），而不是靠主觀判斷。

以前你用 Claude Code 是「給任務，等結果」，碰到問題才回頭調整 prompt。這篇文章提供了一套可重複的設計框架：**任務設計時就考慮驗收標準、context 的持久化位置、工具使用的順序限制**。這不是讓 Claude 更聰明，而是讓你給 Claude 的「工作說明書」更精確。這個思維你其實已經在用——Claude_auto 的 `learning-context.md` 裡的每一個「步驟 + 驗收標準 + 禁止修改聲明」，就是這個原則的落地實踐。

## OpenAI Codex 大升級 — Computer Use、90+ Plugins、MCP 整合、GitHub PR 自動回應

OpenAI 的 Codex（ChatGPT 介面版）在這次大更新中跨越了「coding agent」的邊界。**Computer Use** 讓 Codex 能操控桌面應用；**90+ plugins** 涵蓋 Atlassian Rovo、CircleCI、CodeRabbit、GitLab Issues、Microsoft Suite、Neon、Render 等，大多數透過 MCP 標準接入；最實用的新功能是**直接處理 GitHub PR review comments**——你可以在 Codex 對話裡指向一個 PR，讓它讀取所有 review comments，自動修改程式碼回應每個評論，再提交 commit，整個 PR 迭代流程不需要你自己在 editor 和 PR 頁面之間切換。另外新增 SSH remote devbox 連線（alpha）、多 terminal tab 並行、PDF / 試算表 / 簡報富媒體預覽等。

這次升級的技術關鍵是 Codex 終於全面接上了 MCP ecosystem：90+ plugins 裡的工具都遵循開放標準，讓 Codex 的工具使用不再是 OpenAI 的封閉介面。GitHub PR review comment 自動處理的實作是：Codex 呼叫 GitHub API 拉取所有 review threads，對每個評論理解「reviewer 想要改什麼」，再在對應的程式碼位置做修改，最後生成 commit message 解釋每項修改回應了哪個評論——整個流程是一個完整的 multi-step agent loop，不是簡單的程式碼生成。

以前 Codex 和 Claude Code 有一個關鍵差異：Codex 活在瀏覽器、Claude Code 活在 terminal——這導致開發者要在兩個工作情境之間切換。Codex 的 Computer Use 加入讓它也能介入 terminal 工作流，MCP 的整合讓工具生態系開始趨同。這不是 OpenAI 追上 Anthropic，而是兩家都在往同一個方向聚合。對你最直接的好處是：你在 Nexus MCP Server 上的投資，現在可以同時被 Claude Code 和 Codex 複用——一次開發，多個 AI 工具共用。

## 今日總結

今天的主題是 **Agentic coding 工具的成熟化**：Anthropic 從工程原則層面告訴你怎麼用好 Claude Code，OpenAI 從生態系層面把 Codex 的觸角延伸到整個開發工作流。兩者的共同方向是：AI 工具不再是「聰明的自動完成」，而是「可以被委派完整任務的代理」。

**核心洞察：** Anthropic 的最佳實踐文章揭示了一個在企業系統開發裡特別重要的原則——「讓任務邊界清楚」比「讓模型更聰明」更重要。你在 EQS 系統的流程卡控邏輯，其實就是這個原則的人工版本：每個表單步驟都有明確的進入條件和完成標準。把這個思維帶進 AI agent 設計，是你相對其他開發者的結構性優勢。

**可以應用的行動：**
1. 重新審視 Claude_auto 的 CLAUDE.md，把隱性決策規則改寫成明確原則語句（prefer/always/never 句型）
2. 在 Nexus 的下一個 PR 測試 Codex 的 review comment 自動處理功能
3. 確認 Nexus MCP Server 的工具定義符合 MCP 開放標準，為多工具接入做好準備
