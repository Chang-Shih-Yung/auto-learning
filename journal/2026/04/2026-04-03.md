## Gemma 4 — Google 發布多模態開源模型系列，最大 31B 透過 Ollama 可本地執行

你現在可以在 Mac 或搭載中端 GPU 的本地機器上，用 Ollama 或 LM Studio 跑 Google Gemma 4——最大的 31B 版在 AIME 2026 數學測試拿下 89.2%，agent 工具使用基準（τ2-bench）達 86.4%，同時支援圖片、音訊輸入與 140 種語言。沒有 API 費用，沒有資料上傳。

Gemma 4 是 Google 新世代開源模型系列，分四個尺寸（E2B、E4B、26B、31B），其中 E 系列針對邊緣設備（手機、IoT）優化。架構原生支援函式呼叫（function calling），讓模型能直接觸發工具，而不是只產生文字再靠外部程式解析輸出，這讓 Gemma 4 真正可以嵌入 agent 工作流。最小的 E2B 可在手機上跑，31B 透過 Ollama 在消費級 GPU 上可部署。

以前要跑能用於 agent 的多模態模型，你需要付 OpenAI 或 Google 的 API 費，或租 A100 雲端算力。現在 Google 把同等級的模型直接開源：Ollama 一行指令下載，本地跑，資料不出機器，函式呼叫直接原生支援。對想在自主專案裡整合 AI 又不想依賴外部 API 的場景，Gemma 4 是目前免費選項中最完整的。

## Qwen3.6-Plus — 1M Token 上下文，免費用，比 Claude Opus 快 2–3 倍

你現在可以在 OpenRouter 免費（model string: `qwen/qwen3.6-plus-preview:free`）呼叫 Qwen3.6-Plus——這個模型有 1,000,000 token 的上下文視窗、最多 65,536 token 的單次輸出、原生 function calling，在 terminal 環境的 agentic coding 基準 Terminal-Bench 2.0 拿下 61.6%，比 Claude Opus 4.6 的 59.3% 還高；輸出速度大約是 Claude Opus 的 2–3 倍。整個 Nexus monorepo 的程式碼，可以一次塞進來分析。

Qwen3.6-Plus 來自阿里巴巴 Qwen 團隊，是 3 月下旬釋出的 Preview 版本。它的 1M context 不是靠壓縮或取樣實現的，而是架構層級的設計——hybrid 架構讓它同時保持高速推理和長程一致性。這讓它特別適合「一次性分析整個大型程式庫」的任務，而不是像部分模型那樣需要手動切片。它在 SWE-bench Verified 拿到 72.4%（和 GPT-5 mini 並列）、文件識別 OmniDocBench 91.2%（超過 Claude Opus）。

以前要分析 Nexus 整個 monorepo（packages/ui、packages/cli、apps/demo、apps/www 加起來幾十萬字），你需要分批餵給模型、自己拼接結果，或付高額的 128K context API 費。現在 Qwen3.6-Plus 在 OpenRouter 免費，1M context，一次全塞，輸出速度 2–3 倍於 Claude Opus。全程免費、不需要信用卡，今天就能開始用。

## Claude Code v2.1.91 — MCP 工具回傳上限解鎖到 500K 字元

從 Claude Code v2.1.91 開始，你的 MCP Server 工具回傳結果可以透過 `_meta["anthropic/maxResultSizeChars"]` 標註，把上限從預設值擴大到最多 **500,000 字元**。這個大小足以回傳完整的資料庫 schema、整份元件 catalog、或 Nexus 的完整 registry 定義——不再被截斷。同版本還新增 `disableSkillShellExecution` 設定，可關閉 skills 與自訂 slash commands 內的 shell 執行，讓受管理環境更安全。

原本 Claude Code 對 MCP tool result 有大小限制，超過就靜默截斷，導致 DB schema 這種大型資料回傳到 Claude 時資訊不完整。v2.1.91 引入的機制是讓 MCP Server 在回傳的 `_meta` 欄位裡聲明「這個結果允許更大上限」，Claude Code 才放行。這是 server 端選擇性解鎖，而不是全局關閉限制——保持安全性的同時彈性擴充。另外，multi-line prompts 現在可以透過 `claude-cli://open?q=` deep link 傳入，讓桌面整合更完整。

以前在 Nexus 的 MCP Server 裡設計「回傳完整元件 registry」的工具，很可能超出 Claude Code 的回傳上限，導致 Claude 只看到一半的元件資訊，生成的程式碼缺這缺那。現在加一個 `_meta` 標註，上限擴大到 500K 字元——足夠把所有 60+ 元件的完整 metadata 一次回傳，讓 AI Agent 真正從全局視角操作你的設計系統。

## Ink — Claude Code、GitHub Copilot、Google Gemini CLI 背後都在用的 React CLI 框架

用 Ink，你可以用 React JSX + Flexbox 寫 CLI 工具——不是字串拼接，而是真正的元件、hook、state 管理，輸出到 terminal。你在 Nexus 的 `npx @nexus/cli add button` CLI，可以改用 Ink 加上互動式元件選擇器、即時預覽列表、鍵盤導航——用的是你已熟悉的 `useState`、`useEffect` 語法，外加 Ink 的 `useInput()` hook。這個框架有 50+ 知名使用者，包括 Anthropic（Claude Code 本身）、GitHub Copilot CLI、Google Gemini CLI、Shopify、Linear。

Ink 的核心是把 React 的虛擬 DOM 渲染目標從瀏覽器 DOM 換成 terminal 輸出。它底層用 Yoga（Meta 的 Flexbox 引擎，同時用在 React Native）處理 layout，讓 `<Box>` 元件的 `flexDirection`、`padding`、`gap` 屬性可以直接對應到 terminal 的排版。所有 React hooks 全部相容，第三方 React 狀態管理也可以套用。對 TypeScript 的支援完整——整個框架本身就是用 TypeScript 寫的。

以前寫 CLI 需要學 `chalk`、`inquirer`、`commander` 這些各自獨立的工具，拼湊成一個能用的 CLI 介面——每個工具有自己的 API 風格，不可組合，測試困難。現在 Ink 把 CLI 開發帶進 React 生態：可測試的元件、可復用的 hook、一致的 Flexbox 排版。Claude Code 本身在 production 用它，代表穩定度足夠——你的 Nexus CLI 可以直接套。今天在 GitHub Trending 拿下 305 顆星。

## claude-brain — 用 6 個 Hook + SQLite 讓 Claude Code 不再失憶，67,000 條對話可跨 Session 搜尋

claude-brain 讓 Claude Code 記住所有對話：1,321 個 session、67,000+ 則訊息、27,000+ 個 semantic embedding，全部存在本地 SQLite，透過 11 個 MCP 工具 + 14 個 slash command 供 Claude 查詢。你可以問 Claude「上個月我們在 Nexus 怎麼解決那個 RxJS 訂閱洩漏問題？」——Claude 會從歷史搜尋，而不是說「我不記得之前的 session」。

作者掛了 6 個 Claude Code lifecycle hook（`session-start`、`user-prompt-submit`、`stop`、`session-end`、`pre-compact`、`post-compact`），每次對話自動完整捕獲。搜尋有三種模式：FTS5 關鍵字搜尋（快速精確）、sentence-transformer 語意搜尋（模糊相似）、fuzzy 模糊比對——三種都可以跨 9 個專案查詢。MCP 工具讓 Claude 在新 session 裡直接呼叫搜尋，不需要複製貼上舊對話，還支援自動 email 摘要（daily standup、weekly summary 模板）。

以前每開一個 Claude Code session，就像換了一個新同事——你要重新交代背景、重新解釋架構、重新說明為什麼那個決策這樣做。現在 claude-brain 把 67,000 條對話建索引，每次 Claude 可以主動查上下文，「我記得三週前討論過這個問題，當時的結論是...」——這不是強化 Claude 的記憶，而是給 Claude 一個可以查詢的外部資料庫。一次安裝（Python 3.10+），支援 macOS、Ubuntu、Windows，可選 iCloud 或 Dropbox 雲端備份。

## 今日總結

**本地 AI 全面崛起：** Gemma 4（Ollama 可跑，31B，τ2-bench 86.4%）和 Qwen3.6-Plus（OpenRouter 免費，1M context，比 Claude Opus 快 2–3 倍）同日浮現，代表「本地 / 免費 agent-ready 模型」這條線已經真正可用。以前這是雲端 API 的專屬領域，現在前端工程師在自己的機器上就能跑。
