## Claude Code v2.1.85 正式釋出

**來源：** GitHub Releases · anthropics/claude-code · 2026-03-26

因為你每天用 Claude Code 跑這個學習日誌的自動化流程，這次更新讓你可以直接受益：

- **Hooks 加入 `if` 條件欄位**：不再是「每次操作都觸發 hook」，而是只在特定條件滿足時才跑——對你這種「commit 才觸發 CI」的工作流很實用，能大幅降低無效的 process overhead。
- **Scheduled tasks 加上 timestamp marker**：transcript 裡現在有時間戳記，讓你在除錯學習日誌的定時任務時，可以精確追蹤每個步驟的執行時間點。
- **WASM → TypeScript yoga-layout**：scroll 效能改善，大型 session 的 compaction 不再卡頓——你用 Claude Code 做長對話分析的體驗會更順。
- **MCP 多伺服器環境變數支援**：`headersHelper` scripts 現在支援多個 MCP server 的環境變數，對你未來擴展工具鏈很有用。
- **記憶體洩漏修復**：remote session 中斷串流時的 memory leak 已解決，ECONNRESET 錯誤透過 fresh TCP 重試修復。

## 研究警告：AI 生成程式碼不比人類更安全

**來源：** The Register · 2026-03-26 · Georgia Tech SSLab 研究

因為你用 Claude Code 寫程式，這篇研究的數據讓你必須重新校準對 AI coding assistant 的信任邊界：

- **74 個 CVE 來自 AI 工具**（2025/05 ~ 2026/03）：Claude Code 49 個（11 個 critical），GitHub Copilot 15 個——研究者估計實際數字是追蹤到的 5-10 倍。
- **48% 的 AI 生成程式碼有 bug**：Georgetown 研究顯示，編譯成功的 AI 程式碼中，只有 30% 通過安全驗證。
- **AI 只佔 4% 的 GitHub commits，但貢獻了不成比例的漏洞**：這意味著 AI 生成程式碼的「危險密度」遠高於平均。

**對你的意義**：CI/CD 裡的 SAST 工具不是可選項，而是在 AI 輔助開發時代更加必要的防線。

## Show HN: Axe — 12MB binary，用 Unix pipeline 哲學取代 AI frameworks

**來源：** Hacker News · github.com/jrswab/axe · 227 points

因為你懂 CI/CD pipeline 設計，Axe 的設計哲學會直接打中你：

- **12MB Go binary，沒有依賴**：不是另一個 Python agent 框架，是一個把 LLM 當 Unix 程序對待的工具。
- **stdin piping + sub-agent delegation**：LLM 呼叫可以透過 pipe 串連——`cat prompt.txt | axe gpt4 | axe claude` 這樣的模式讓你把 AI 工具整進現有 shell workflow。
- **Path-sandboxed file operations**：agent 的檔案存取被限制在指定路徑，防止意外修改範圍外的檔案。
- **不是 framework，是 composable tool**：和 LangChain / CrewAI 等框架相比，Axe 的哲學是「composability over monolithic」。

## 今日總結

今天最值得記住的三個洞察：

**1. 工具成熟 ≠ 可以降低警戒（AI 程式碼安全研究）**
Georgia Tech 的數據提醒你：AI coding assistant 讓你更快，但不讓你更安全。在 CI/CD pipeline 裡保持 SAST 不是舊習慣，而是新時代裡更必要的防線。因為你已懂 SAST，下一步是在 pipeline 中加入 AI 生成程式碼的專項規則。

**2. MCP 從「工具」變成「基礎設施」（97M installs 里程碑）**
當一個協議有 4,000+ 伺服器和官方安全標準時，它就不再是「你可以選擇要不要學的東西」——而是和 Docker 一樣的基礎設施知識。你每天用的 Claude Code 已經是這個生態的一部分；MCP Security Standard v1.1 是下一個值得研讀的文件。

**3. 輕量優先於框架（Axe 的 Unix 哲學）**
Axe 的出現是一個訊號：開發者開始對 agent 框架的重量感到疲憊。「把 LLM 當 Unix 程序」這個思路，和你已有的 pipeline 直覺完全相容——在你下次設計 AI 工作流時，先問「這能不能用 pipe 做到」，再決定是不是需要 framework。
