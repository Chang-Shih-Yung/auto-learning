# 2026-03-30 學習日誌

## Figma MCP Server 正式開放寫入：Claude Code 可以直接在 Figma Canvas 上操作元件

你現在可以用 Claude Code 直接在 Figma 裡建立和修改元件——指定你的設計系統作為 context，讓 agent 自動建立符合規格的 frames、variables 和 auto layout，不需要手動在 Figma 裡拉元件。開啟 Code Connect 之後，agent 輸出的不是通用 React 程式碼，而是呼叫你自己的元件庫、你的 import 路徑、你的 prop interface。這意味著設計系統的 single source of truth 從單方向（Figma → code）變成雙向：code 端的更新可以反推回 Figma canvas。Figma MCP beta 期間免費開放，支援 Claude Code、Cursor、GitHub Copilot、Warp 等主流開發工具。

這背後是 Figma 給 AI agent 授予了寫入 API 的存取權，讓它能操作 Figma 的 native 物件模型——frames、components、variables、auto layout 都在 agent 可控制的範圍內。Code Connect 的關鍵在於把每個 Figma 元件連結到對應的程式碼元件：agent 讀圖時看到的不是「一個按鈕形狀」，而是「Button component，接受 `variant: primary | secondary` 這個 prop」。這讓 agent 的輸出直接符合你既有的設計語言，省去事後手動對齊的往復。

以前的 Figma → code 工作流是單行道：設計師在 Figma 畫好，工程師再翻譯成程式碼，設計系統的「真相」永遠停留在 Figma 端。現在這條路可以雙向走：工程師在 code 端調整元件規格，agent 透過 Figma MCP 同步更新 Figma 裡的設計稿，兩端不再需要人工手動同步。對有大量設計 token 和元件規格的專案來說，這不只是省時，而是消除了「Figma 版本和 code 版本不一致」這個長期存在的協作摩擦點。

## Cloudflare Turnstile 已能讀取 React 內部狀態：bot detection 進入應用程式執行層

如果你在 React SPA 裡實作 bot protection，現在有一個新的技術標準可以參考：Cloudflare Turnstile 在 ChatGPT 部署了一套 55 屬性、三層偵測系統，其中第三層直接讀取 `__reactRouterContext`、`loaderData` 和 `clientBootstrap`——這些只有在 React 完整 hydration 之後才存在的內部狀態。換句話說，只模擬瀏覽器 API 的爬蟲無法通過，因為它們跳過了 JavaScript 執行，根本產生不了這三個屬性。

一位研究者逆向解密了 Turnstile 的混淆程式碼——加密方式是 XOR 搭配嵌入在同一個 bytecode payload 裡的金鑰，本質上是混淆而非真正加密——並成功解開了 377 個樣本，確認每個樣本採集的 55 個屬性完全一致（Layer 1：38 個瀏覽器特徵，包括 GPU、螢幕尺寸、字型、DOM 能力；Layer 2：5 個 Cloudflare edge headers，注入地理位置與 IP 資料；Layer 3：3 個 React 內部參照）。bot detection 的偵測目標已從「瀏覽器是否存在」升級為「你的 React app 是否真的跑起來了」。

以前的 bot detection 在「瀏覽器特徵」層面運作：偽造 user-agent、模仿 GPU 資訊就能過關。現在最先進的防護已移到「JS 執行狀態」——爬蟲必須真的跑完整個 React hydration cycle，才能通過 Turnstile 的第三層檢查。對前端工程師的實際意義是：你的 SPA 的執行痕跡本身就是一種驗證依據。任何「快取 HTML 不跑 JS」的輕量爬蟲策略，在這種防護面前都失效了。

## MCP 協議 2026 里程碑：9,700 萬月下載、OAuth 2.1、Async Tasks 正式上線

你現在可以在你建的 MCP server 裡加入 async tasks 支援——讓 agent 啟動一個長時間工作流之後立刻回應「已開始執行」，任務完成後再推送結果，不需要讓呼叫端等候 blocking response。認證這邊也有大更新：新版 MCP 用 Client ID Metadata Documents（CIMD）取代了原本的 Dynamic Client Registration（DCR），client 的身份變成一個 URL，不需要維護一個不斷增長的 registration 資料庫。目前 Python + TypeScript SDK 合計月下載量達 9,700 萬次，MCP Registry 已有近 2,000 個 server 條目。

MCP 在 2025 年 11 月加入 Linux Foundation 旗下的 Agentic AI Foundation，從 Anthropic 的開源工具正式升格為跨廠商協議標準。Claude、ChatGPT、Gemini、GitHub Copilot、Cursor 現在都在同一個 spec 下運作，意味著你建一個 MCP server 等同於同時對接所有主流 AI 工具。OAuth 2.1 + PKCE 解決了「誰被授權存取哪個 server」的問題；session-scoped authorization 確保長任務的存取權在任務結束後自動失效，不留下永久性的寬鬆授權。

以前你建 MCP server 是為了串接某一個 AI 工具（通常是 Claude）——現在你建的是整個 AI agent 生態系的接口。對 Nexus copilot-sdk 的 MCP server 來說，async tasks 意味著 agent 可以處理「跑完 60 個元件的 SAST 掃描再回報」這種工作，而不是卡在一個 30 秒的 blocking request 上。2026 年的下一步是 enterprise readiness：MCP server 的存取可以整合進公司的 SSO 系統，讓 IT 管理員從同一個 identity provider console 管理 AI agent 的授權範圍。
