## HumanLayer — 讓 AI Agent 在高風險操作前自動暫停問你

你在 Claude Code 或自己的 AI agent workflow 裡，只要加一個 `@require_approval()` 裝飾器，agent 在執行高風險操作（發 email、merge PR、跑 DB migration）前就會自動暫停，透過 Slack 或 email 送出審批請求，等你點「Approve」才繼續。整套流程有完整 audit trail，目前支援 Slack、email、SMS、WhatsApp 四個通知管道，SDK 同時有 Python 和 TypeScript 版本，`pip install humanlayer` 或 `npm install humanlayer` 即可開始。

背後的機制是把「工具呼叫」轉換成「待批准任務」。HumanLayer SDK 攔截 agent 要呼叫某個函數的請求，把它序列化後推送到你設定的管道，讓 agent 進入等待狀態。等你批准後，SDK 把結果注入回 agent 的執行上下文，agent 繼續往下跑——整個過程對 LLM 來說只是一個普通的工具回傳值。這就是為什麼它能做到 framework-agnostic：不管你用 LangChain、CrewAI 還是直接呼叫 Anthropic API，原理完全一樣。

以前你寫 AI agent 要嘛完全自動（風險太高）、要嘛全程人工監控（等於沒自動化）。HumanLayer 讓你可以設計「95% 自動、5% 關鍵節點人工確認」的工作流：agent 自己判斷何時需要你介入，不需要你坐在旁邊盯著跑。YC F24 校友、$500K 預種子輪，基礎版開源免費。

## Magnitude — 7 行自然語言取代幾十行 CSS Selector 測試

你可以用 7 行自然語言描述（例如「新增一個 todo 叫 Buy groceries，然後標記為完成」）取代傳統 Playwright 需要幾十行 CSS selector 的 E2E 測試。Magnitude 在背後用視覺 AI agent 操作你的 app——它看螢幕截圖、決定點哪裡，就像真實用戶一樣，不依賴 DOM 結構。`npm install --save-dev magnitude-test`，本地跑或整合進 CI/CD pipeline 都支援。

架構是雙 agent 系統：Planner（Gemini 2.5 Pro 或 Claude 等強模型）負責把自然語言指令轉成操作計劃；Executor（Moondream 2B，輕量開源視覺模型）負責在截圖上定位精確像素座標並執行點擊。Planner 生成的計劃可以快取，後續只跑 Executor——速度快、成本低。遇到 UI 變動時，Executor 回報失敗，Planner 重新規劃，不再因 DOM 結構改變就整批炸掉。

以前你重構 Angular component 的 DOM 結構，Playwright 的 `data-testid` selector 就可能失效，需要手動更新測試。現在 Magnitude 的測試看的是畫面截圖而非 HTML，重構之後只要視覺結果一樣，測試就通過。這直接解決了 Nexus 元件庫 60+ 元件測試覆蓋率背後最大的維護痛點——selector 漂移。

## Chroma v1.5.5 — Rust 重寫後 4 倍速提升，本地跑 Zero Network Latency

你可以在 MacBook 上用一行 `pip install chromadb` 啟動嵌入式向量資料庫，讓 AI agent 或 RAG pipeline 做語意搜尋時完全不需要網路呼叫——zero network latency。Chroma v1.5.5（2026-03-10 發布）Rust 重寫後，寫入和查詢速度比原本 Python 版快 4 倍，同時支援向量搜尋、全文搜尋、regex 搜尋三合一，不需要另外整合 Elasticsearch。

Rust 版核心是 push-based morsel-driven execution engine，加上 base64 向量編碼優化。從 v1.5.0 開始，系統資料庫（sysdb）整個用 Rust 重寫，collection 的創建和查詢路由都跑在 Rust 層，Python 只是最外層的 API wrapper。嵌入式模式下，Chroma 跑在你的 application process 裡，省去 TCP 握手和序列化的開銷——這就是速度比雲端服務快的原因，也是「本地 first」策略能成立的底層基礎。

以前要給 AI agent 加語意搜尋能力，你要嘛租 Pinecone 的雲端向量資料庫（有 API key、費用、資料外洩疑慮），要嘛自己架 server。現在 Chroma embedded 模式，一行 import，資料庫跑在你的 code 裡，向量留在本地機器。對 Nexus Copilot SDK 來說，這意味著你可以讓 AI 元件推薦功能做語意比對，無需任何外部依賴。

## CSS is DOOM — 用純 CSS 3D 渲染 1993 年的射擊遊戲

你可以在瀏覽器裡用純 CSS 跑 DOOM——每面牆、每個地板、每個怪物都是一個 `<div>`，用 CSS 3D transforms 定位在三維空間中。不需要 WebGL，不需要 Canvas，CSS 的 `hypot()`、`atan2()`、`sin()`、`cos()` 數學函數加上 `clip-path` 和 `@property` 撐起整個 3D 引擎。Safari on iOS 會因為要同時處理幾千個 3D transform 元素而 crash——但這件事本身就說明了 CSS 渲染架構正在被逼到極限。

Niels Leenheer 用 CSS `translate3d()` + `rotateY()` 重建 DOOM 的幾何結構。牆的寬度用 `hypot()` 計算（畢氏定理），旋轉角度用 `atan2()` 求出。`@property` 讓 `--player-z` 這類自訂屬性可以做 CSS 原生的 smooth transition，不需要 JS 介入。`clip-path: polygon()` 切割 sector 邊界，SVG filter（`feColorMatrix` + `feTurbulence` + `feDisplacementMap`）製作幽靈的隱形效果。整個 3D 引擎的數學計算全部用 CSS 表達式完成，沒有一行 JS 做渲染運算。

以前瀏覽器 3D 渲染都要靠 WebGL 或 Canvas，CSS 只是用來排版。現在 CSS 的數學函數加上 `@property` 讓 custom properties 成為可以動畫的計算節點——這代表你建立的設計 token 系統不只是靜態值，而是可以直接接受 `transition` 的動畫參數。從 OKLCH 色彩切換到 loading 動畫，同一套 CSS Custom Properties 架構就能打通。

## 今日總結

**Agent 的可控性是工程問題，不是信任問題。** HumanLayer 的核心洞察——不是你不信任 AI，而是「讓高風險操作有明確的 human checkpoint」本身就是好的系統設計。這跟 EQS 前端流程卡控的邏輯完全對稱：你卡使用者，HumanLayer 卡 AI。

**測試的未來是視覺而非 DOM。** Magnitude 的雙 agent 架構（Planner + Executor）讓測試從 selector 地獄中解脫——視覺 AI 看的是截圖，不是 HTML。DOM 重構不再是維護噩夢，7 行自然語言取代幾十行 selector 不是噱頭，是架構決策。

**AI 基礎設施往本地收斂。** Chroma Rust 重寫 + embedded 模式代表一個趨勢：向量資料庫開始認真對待「本地 first」——4x 速度提升 + zero network latency，讓開發者不需要為了 MVP 就去租雲端服務，也讓資料不需要離開本機。

**CSS 已經是可程式化的計算引擎。** `@property` 讓 custom properties 成為動畫節點，`hypot()` 和 `atan2()` 讓 CSS 做出幾何計算。對 Nexus token 系統的啟示：設計 token 不只是靜態值，是可以被 transition 的動畫參數。

**本週應用點**：在 Nexus Copilot SDK 的 agent workflow 裡加入 approval gate（HumanLayer 概念），考慮用 Chroma embedded 作語意記憶後端，用 Magnitude 替換部分 E2E selector 測試，研究 `@property` 讓 OKLCH token 支援 smooth transition。
