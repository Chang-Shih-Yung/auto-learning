## Claude Code v2.1.88：PermissionDenied Hook + Compound Command 修復全覽

Claude Code v2.1.88（2026-03-30 發布）帶來兩類更新——新 API 和品質修復。新 API：`PermissionDenied` hook 讓你在 auto mode 拒絕操作後回傳 `{ retry: true }` 自動重試、`CLAUDE_CODE_NO_FLICKER=1` 消除 alt-screen 閃爍、`@` 補全現在包含命名 subagent 清單。修復：compound command（例如 `git add && git commit`）的 hook 條件過濾誤判問題修好了，長 session 的 prompt cache miss（因 tool schema bytes 變動）也一併修復，長時間 unattended run 更穩定。

`PermissionDenied` hook 讓你在 auto mode 的「審核閘口」掛自訂邏輯——讀取白名單檔案、判斷被拒絕的操作是否符合規則、符合就回傳 retry。這把原本二元的守門員升級成帶條件的審核層。Compound command hook 修復則讓「指定特定指令才觸發」這件事行為可預期，不再有多餘觸發。

以前，auto mode 遇到未預期操作就中斷等人確認，compound hook 會誤觸發無關指令；現在，`PermissionDenied` hook 讓排程任務自我決策，compound hook 精準到單一指令層級——兩個修復加在一起，讓 unattended run 的可預測性大幅提升。

## Cursor 2.5：8 個並行 Agent 在隔離 VM 跑，BugBot Autofix 35% 直接 merge 進主分支

在 Cursor 2.5 裡，你可以同時啟動最多 8 個 AI agent，每個都跑在獨立的 Ubuntu VM 上，帶自己的 Git worktree——修改互不干擾、context 互不污染，任務結束後環境自動回收。BugBot 從「只出報告」升級為「找到問題、自己開 VM、測試修復、提 PR」，超過 35% 的 BugBot Autofix 被直接 merge 進主分支，不是玩具，是進入生產的自動修復流程。

這背後是 Cloud Agents 架構：每個 agent 拿到環境副本、有獨立的 CLI 存取和 Git worktree，coordinator 只負責分派 task，不共享中間結果。這是 2026 年主流 AI coding tool 解決 context pollution 的標準做法。同時推出的 MCP Apps（v2.6）讓 agent 在對話框裡直接渲染互動式圖表和白板——把「看 AI 輸出」從純文字升級成可操作的視覺介面。

以前你要同時跑多件事，只能開多個 terminal session 手動 stash 切換；現在 Cursor 的 8-VM pool 把這個協調層做掉了。BugBot 的 35% merge rate 意味著超過三分之一的 PR 問題可以在進入 code review 之前就自動解決——對 Monorepo 這種跨 package 改動頻繁的專案，人工 code review 的初篩成本接近歸零。

## Long-Horizon Agents：supervised 工作流是真實可用的，全自動是假設

如果你的 agent 有 CLI 存取、可讀 log、能跑測試、任務相互獨立——你現在已經可以讓它執行有意義的長任務，不需要等什麼「真正的 AGI」。BugBot Autofix 的數字說得很清楚：35% 的 AI 提出的 PR 修復被直接合併，但剩下的 65% 裡有相當比例是 agent 通過了測試卻沒有達成原始意圖——narrow verifier 過了、使用者感受沒到位。這是當前 long-horizon agent 的邊界。

軟體之所以是 AI agent 最好的實驗場，是因為軟體可測、可逆、可驗證：agent 可以直接執行測試確認輸出是否正確，不需要人在中間做模糊判斷——這個特性在其他領域幾乎不存在。有效的 long-horizon agent 模式是：給 CLI 存取、給文件存取、加 preflight check、跑 headless test，讓 agent 在 feedback loop 裡自我修正。複雜的多 agent 階層協調（manager → 多層 worker）目前仍然脆弱，simpler planner/worker 結構在多次測試中勝過複雜方案。

以前你的直覺是「讓 AI 跑，出錯再介入」；現在正確的期待是：你是 demanding QA，不是旁觀者。agent 在跑、你隨時可以介入，但不需要逐步下指令——這個 supervised autonomy 姿態，才是 2026 年 AI agent 工作流的實際型態。全自動的誘惑很大，但 35% merge rate 告訴你：另外 65% 需要你的判斷。

## Ollama 0.19：MLX 加持，Apple Silicon 本地跑 35B 模型速度近翻倍

如果你的 Mac 有 32GB 以上 Unified Memory，現在可以在本地跑 Qwen3.5-35B-A3B——prefill 速度 1,810 tokens/秒（比上一版 1,154 快了 57%），decode 速度 112 tokens/秒（比上一版 58 快了 93%）。20,000 字的 codebase 幾秒掃完，全程在你自己的機器上運算，不需要任何雲端訂閱。

這背後是 Apple MLX framework 的功勞——一套專為 Apple Silicon 統一記憶體架構設計的機器學習框架。Ollama 0.19 同時導入 NVFP4 量化格式（壓縮模型體積、幾乎零精度損失）和智慧 cache checkpoint（推理中途存快照，下次對話直接接續），讓同樣的記憶體換來近兩倍的吞吐量。

以前，在本地跑 35B 等級模型需要配大顯卡的工作站，或是把資料上傳到雲端 API。現在，同樣的推理能力直接跑在 MacBook Pro 上——一次下載、永久離線、資料不外洩、沒有訂閱費。Ollama 0.19 目前以 preview 身份釋出，需要 32GB+ Unified Memory；更多模型架構的支援正在開發中。

## 緊急安全警報：axios NPM 供應鏈遭攻擊，v1.14.1 和 v0.30.4 含惡意 RAT

如果你的專案安裝過 axios@1.14.1 或 axios@0.30.4（2026-03-30～31 間短暫上架），立刻固定版本回 axios@1.14.0 或 0.30.3，並完整移除 plain-crypto-js。axios 全球每週下載量超過 3 億次，受影響系統需視為完全淪陷、全面輪換所有 credentials（npm token、cloud key、CI/CD secret）。

攻擊者透過入侵一名維護者帳號，悄悄在兩個版本裡塞進隱形依賴 plain-crypto-js@4.2.1。這個套件從未出現在 axios 實際程式碼中，卻在 npm install 時靜默執行 postinstall 腳本，下載跨平台 RAT 並向 sfrclak.com:8000 的 C2 server 回報——完成後自我替換成乾淨的 package.json，抹去所有痕跡。整個攻擊鏈從安裝到植入只需幾秒，且靜默無聲。

以前，依賴知名套件（3 億週下載量）搭配 package-lock.json 鎖版，就算是合理的供應鏈防護。現在，一個維護者帳號被入侵就足以讓整個生態系暫時中毒。對所有前端工程師的新底線：CI/CD pipeline 加上 `npm ci --ignore-scripts`，讓 postinstall 腳本永遠無法在自動化流程裡執行。這一行設定耗時不到一分鐘，卻能擋住這整類攻擊。

## Universal Claude.md：一個設定檔讓 Claude 輸出 token 減少 63%

把一個特製的 `CLAUDE.md` 檔放進專案根目錄，每次 Claude 的回應平均從 465 字壓縮到 170 字——output token 減少 63%。如果你每天跑 1,000 次 Claude API 呼叫（例如 agent loop），每月省下約 $8.64；跑 10,000 次則省 $86——不需要改任何程式碼，只需要一個文字檔。

這個技術叫「行為約束型 prompt engineering」——不是修改模型，而是透過 CLAUDE.md 寫入六條強制規則：禁止以「Sure!」「Great question!」開頭、禁止在回覆前重述問題、強制輸出 ASCII（避免 em dash 破壞 parser）、禁止不必要的免責聲明、限制回覆範圍不超出實際需求、以及遇到不確定時直接說「I don't know」而非猜測。這六條規則刪掉的都是 Claude 預設愛加的「社交潤滑劑」，對技術任務完全無用。

以前，Claude 在高頻 agent loop 裡每次回應都夾帶 60%+ 的廢話 token——「I'll help you with that!」「Here's what I found:」「I hope this helps!」——這些字符不只花錢，還佔用 context window，讓後續 agent step 需要處理更長的 history。現在，一個 CLAUDE.md 讓每次輸出直接進入正題，agent loop 更快、context 更省、成本更低。注意：此方法的 input token 略增（CLAUDE.md 本身需要讀入），適合高頻批次呼叫，低頻單次查詢不划算。
